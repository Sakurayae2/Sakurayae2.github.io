搭建Hexo博客

· 2021-8-18 · 次阅读

       最近心血来潮花了点时间在网上到处搜刮资料搭建了这个博客，苦于肚子里没什么墨水儿也没有干货，就想着把这两天搭建这个博客的过程写下来，水几篇文章（bushi）, 哦是记录下来，怕到时侯换设备什么的忘了又得重头再来，顺便练习下 markdown 语法。（这才是你真正的目的吧！）

ps: 过程中有什么问题建议或者有什么纰漏都欢迎留言（当然了你没事儿逛逛也可以说点什么嘛），那什么第一次写文章有点。。。将就点看吧哈哈q(≧▽≦q)

1.前期准备

1.1.安装 Node

1.点击前往Node官网下载并安装;
然后验证下node是否安装成功,打开cmd输入以下内容:

#有版本信息就说明安装成功
    node -v   #查看node版本
    npm -v    #查看npm版本

2.npm换源,接着在cmd中输入:

#因为众所周知的原因,还是换个国内的管理工具快点
    npm install -g cnpm --registry=https://registry.npm.taobao.org

再来验证下cnmp是否安装成功：

    cnpm -v

1.2.安装 Hexo

还是在cmd中全局安装hexo:

    cnpm install -g hexo-cli

2.创建博客

2.1.创建目录

在D盘根目录下新建一个blog文件夹,然后cd进blog目录:

#你们可以随自己建文件,我这里就建在D盘根目录了
    cd D:\
    mkdir blog
    cd \blog

2.2.初始化

在刚建立的博客目录下,初始化你的博客:

    hexo init
    #初始化后你的博客目录下应该已经有了一堆乱七八糟的东西，那恭喜你已经成功了一半

2.3.启用服务

启用本地博客服务:

    hexo s

打开浏览器,输入 localhost:4000 即可在本地查看你的博客,默认写有一篇Hello World文章(万物皆从Hello World起)。

3.撰写博客

依旧是在博客目录下,cmd:

    hexo n '我的第一篇博客文章'   #名字随意

然后清理生成启动一条龙:

    hexo clean && hexo g && hexo s

回到浏览器,刷新即可看到新建的博客文章。

4.部署博客

4.1.新建仓库

登陆github,没有的话就注册一个,然后点击头像旁的 + 号,选择新建仓库(New repository),注意仓库名(Repository name)的固定写法是: 你的名字.github.io (比如我的就是 Sakurayae2.github.io),除了名字不一样外必须是这个格式,这就是你以后网站的地址

4.2.安装插件

    cnpm install --save hexo-deployer-git

4.3.修改文件

找到blog根目录下的 _config.yml 文件,用记事本什么的打开,
拉到最底下,修改里面 #Deployment 下的内容:

deploy:
    type:'git'
    repo: https://github.com/YourGithubName/YourGithubName.github.io.git  #你的仓库名称
    branch: master  #不写默认创建master分支,而如果你master里面已经有内容了,也可以创建个别的分支(名字随意)

4.4.部署

回到cmd,输入:

    hexo d

打开浏览器,输入你的网站地址,如果你到这儿能看到博客内容(可能要刷新一会儿),那么恭喜你已经成功搭建了一个博客。

当然即使失败了也不要紧,把目录删了重头再来就是,如果你遇到什么问题可以留言问我或者点击下方的链接去看这位大佬的视频教学(我也是看这个学的)
